## docs/overview.md

# Overview

Namazu API is the external integration layer for the Namazu ecosystem.

Its role is to connect:
- mobile applications
- smartphone and IoT sensor inputs
- cloud-linked building workflows
- future analytical modules

The platform should be understood as a structured interface for controlled external collaboration.

## Core concept

Namazu API is intended to make existing and future components explainable through a common interface.

This includes:
- reference mobile-client behavior
- media delivery
- sensor data intake
- building-linked workflow retrieval
- future tensor / TGL-based analytical expansion

## Why this matters

A platform of this type becomes more useful when it can be explained and adopted in a controlled manner.

That is particularly important when engaging with:
- ministries and public-sector institutions
- universities and research organizations
- enterprises operating in building, infrastructure, or resilience-related fields

## Strategic principle

The first public interface should remain modest and practical.

The objective is not to publish every internal capability at once.

The objective is to establish:
- a credible external face
- a pilot-ready integration layer
- a path toward deeper building and analytical workflows
