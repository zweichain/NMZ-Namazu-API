# Use Cases

Namazu API is designed to support controlled use cases across public, research, and enterprise settings.

## 1. Reference mobile application

A reference application such as ZweiMedia can:
- retrieve curated media content
- present ecosystem information to end users
- capture smartphone motion or acceleration data
- upload data into the Namazu cloud workflow layer

## 2. Building-linked workflow integration

A building-related workflow can:
- receive sensor-linked records
- associate them with a building identifier
- retrieve processing status or latest result information
- bridge field data to a dashboard process

## 3. Ministry or municipality pilot

A ministry or municipality may use Namazu API as a structured intake and workflow layer for:
- disaster-prevention pilots
- field-linked data intake
- resilience-related operational trials
- future building-related analytical collaboration

## 4. University or research collaboration

A research organization may use Namazu API to:
- receive or submit structured sensor-linked records
- connect field data to model-oriented processes
- prepare for future TGL-based analytical workflows

## 5. Enterprise application partner

A selected partner application may use Namazu API to:
- provide curated content to users
- submit motion or acceleration records
- retrieve building-linked status or result information

## 6. IoT vendor integration

An IoT vendor may use Namazu API to:
- send device-generated sensor data
- link records to building or project identifiers
- prepare for future analytical or dashboard-oriented integration
