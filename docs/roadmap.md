# Roadmap

Namazu API should be developed in stages.

## Phase 1 — Public foundation

Goal:
- establish a credible public explanation layer
- define the first API surface
- support initial institutional and partner discussions

Deliverables:
- README
- overview and use-case documents
- OpenAPI starter definition
- example payloads
- reference architecture

## Phase 2 — Controlled partner pilots

Goal:
- support initial integration trials
- validate operational assumptions

Deliverables:
- staging environment
- partner credential management
- request logging and validation rules
- partner examples

## Phase 3 — Building workflow expansion

Goal:
- deepen the building-oriented operational value

Deliverables:
- richer building metadata
- floor-linked association
- result history
- stronger dashboard linkage

## Phase 4 — Namazu TGL integration

Goal:
- connect field records to future analytical processes

Deliverables:
- analytical request formats
- model-related interface definitions
- interpretable result structures
- research collaboration paths

## Phase 5 — Contribution / verification expansion

Goal:
- support future participation, record, and verification logic

Deliverables:
- contribution record structures
- participation-related workflow design
- future reward / verification interfaces

## Strategic note

At every phase, Namazu API should be presented first as:
- a disaster-tech platform
- a building-tech platform
- a mobile-to-cloud workflow platform
- a future analytical integration platform
