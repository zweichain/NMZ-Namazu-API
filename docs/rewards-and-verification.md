# Rewards and Verification

This document is reserved for future participation, contribution, and verification-oriented workflows.

## Current stance

The contribution / verification layer may become important in the broader ecosystem, but it should not dominate the initial public explanation of Namazu API.

## Recommended positioning

Present this as:
- a future extension layer
- a participation-related workflow possibility
- a structured expansion path rather than the first public focus

## Why this matters

For ministries, universities, and enterprise partners, the platform’s operational and public-interest value should remain primary.
